Sass - Syntactically Awesome Style Sheets

What is Sass?

It’s a CSS preprocessor, or a stylesheet language that compiles to CSS. It allows you to to keep things organized and create stylesheets more efficiently using features such as variables, nested rules, operators and more. It supports all versions of CSS.

Installation

1) Standalone
- Download package from https://github.com/sass/dart-sass/releases/tag/1.26.3
- Unzip and add folder to PATH
2) Node.js (npm) - (Disclaimer: runs somewhat slower than other options):
npm install -g sass
3) Chocolatey (Windows) - (https://chocolatey.org/install):
choco install sass
4) Homebrew (Mac):
brew install sass/sass/sass

Compiling

To manually compile your Sass file into CSS, type in the console:
sass input.scss output.css
To have it re-compile into CSS each time you save your Sass:
sass --watch input.scss output.css
To have it watch for all files in one directory (ex. app/sass) and compile CSS to another (ex. public/stylesheets):
sass --watch app/sass:public/stylesheets

Variables

Variables work the same way you’d expect them to.

Sass CSS
$base-color: #c6538;
$border-dark: rgba($base-color, 0.88);

.alert {
border: 1px solid $border-dark;
} 		.alert {
border: 1px solid rgba(198, 83, 140, 0.88);
}

Operators, Conditionals, and Loops

- Standard math operators: + , - , * , / , and %
- Comparative (>,<,==,>=,<=, !=) and boolean (and, or, not) operators
- Conditionals and loops (if, else, for, while, each)

Sass CSS
@mixin button-format( $round-button, $size ) {
color: white;
background-color: blue;
width: $size;

@if $round-button and $size > 50px {
height: $size;
border-radius: $size / 2;
}
}

.mybutton {
@include button-format(true, 100px);
} 		.mybutton {
color:white;
background-color: blue;
width: 100px;
height: 100px;
border-radius: 50px;
}

Mixins

A mixin lets you make groups of CSS declarations that you want to reuse throughout your site. You can even pass in values to make your mixin more flexible.

Sass CSS
@mixin transform ( $property ) {
-webkit-transform: $property;
-ms-transform: $property;
transform: $property;
}
.box { @include transform(rotate(30deg)); } 		.box {
-webkit-transform: rotate(30deg);
-ms-transform: rotate(30deg);
transform: rotate(30deg);
}

Nesting

Sass lets you nest your CSS selectors in a way that follows the same visual hierarchy of your HTML

Sass CSS
nav {
ul {
margin: 0;
padding: 0;
list-style: none;
}

li { display: inline-block; }

a {
display: block;
padding: 6px 12px;
text-decoration: none;
}
} 		nav ul {
margin: 0;
padding: 0;
list-style: none;
}
nav li {
display: inline-block;
}
nav a {
display: block;
padding: 6px 12px;
text-decoration: none;
}

Modules and Partials

You can split up your Sass code into separate files and load them as modules, which means you can refer to its variables, mixins, and functions in your Sass file with a namespace based on the filename. Partials are files containing snippets of CSS not meant to be compiled into their own CSS files, but included in other Sass files (indicated by a ‘_’ before the filename).

Sass CSS
###_base.scss file###
$font-stack: Helvetica, sans-serif;
$primary-color: #333;

body {
font: 100% $font-stack;
color: $primary-color;
}

###Primary Sass file###
@use 'base';

.inverse {
background-color: base.$primary-color;
color: white;
} 		body {
font: 100% Helvetica, sans-serif;
color: #333;
}

.inverse {
background-color: #333;
color: white;
}

* Sass has some of its own built-in modules for use as well (https://sass-lang.com/documentation/modules)

Extend/Inheritance

Lets you share a set of CSS properties from one selector to another.

Sass CSS
.foo {
color: black;
border: 1px solid black;
}

.bar {
@extend .foo;
background-color: red;
} 		.foo, .bar {
color: black;
border: 1px solid black;
}

.bar {
background-color: red;
}
